## 1. 기획

### ■ 설정한 사용자는 누구이며, 해결하려는 구체적인 문제점/불편함은 무엇인가요?

**[ 대상 사용자 ]**

국내 대학교에서 학기 단위 팀 프로젝트(팀플)를 반복적으로 수행하는 대학생 전반. 학기마다 새로운 팀이 구성되며 역할이 유동적인 환경에서 학습하는 학습자를 핵심 타겟으로 설정한다.

**[ 핵심 페인 포인트 ]**

| # | 페인 포인트 | 구체적 상황 |
|---|---|---|
| 1 | 역할 불명확 | 팀 구성 직후 누가 무엇을 맡을지 정하는 데 1~2시간 소요, 갈등 발생 |
| 2 | 무임승차 | 기여도 측정 수단이 없어 불공정한 결과가 학기마다 반복됨 |
| 3 | 툴 진입 장벽 | Notion, Jira 등 기존 협업 툴은 2~8주짜리 팀플 대비 세팅 비용이 지나치게 큼 |
| 4 | AI 활용 공백 | 팀플에 AI를 적극 활용하고 싶지만 어떻게 녹여야 할지 알 수 없음 |

**[ 기존 솔루션의 한계 ]**

Asana AI Teammates, ClickUp AI, Monday.com AI 등 기존 AI 협업 툴은 전부 기업·직장인 대상으로 설계되어 있다. 학생 팀플은 기간 2~8주, 역할 유동적, 무임승차가 최대 고통이라는 점에서 기업 툴이 이 맥락을 해결하지 못하며, 이를 전용으로 지원하는 도구가 국내에 존재하지 않는다.

---

### ■ 문제를 해결하기 위한 솔루션의 핵심 기능은 무엇인가요?

**[ 솔루션 개요 ]**

Teamver는 학생 팀플 전용 AI 협업 설계 도구다. 팀 구성 → 역할 설계 → 진행 추적 → 기여도 시각화까지 하나의 플로우로 완결하며, 채팅·일지·마일스톤 전반에 AI를 자연스럽게 녹여 팀플 경험을 개선한다.

**[ 기능 1 ] AI 킥오프 설계자 (도메인 페르소나 시스템)**

- 팀원 이름·스킬셋·성향·프로젝트 목표·기간 입력
- 프로젝트 주제(게임·마케팅·앱·AI 등)를 자동 감지하여 9개 도메인별 전문 페르소나 적용
- Gemini가 최적 역할 분배안 + 주차별 마일스톤 자동 생성
- 이전 킥오프·집계·일지를 참조하여 재실행 시 팀 이력 반영
- 기획안 파일을 업로드하면 AI 킥오프 시 자동 참고

**[ 기능 2 ] 기여도 추적 및 AI 집계 에이전트**

- 팀원이 매일 완료 작업·메모·달성률을 직접 입력
- **AI 집계 에이전트**: 최근 30일 기여 로그·마일스톤·파일·채팅을 종합 분석
  - 팀 건강도(good/warning/critical) 진단
  - 리스크 감지(high/medium/low)
  - 팀원별 기여도 점수(0~100, 가중치 기반) 및 우선순위 제안
- 기여 기록 수정 이력(히스토리 스냅샷) 보존

**[ 기능 3 ] 팀 일지 시스템**

- **AI 내용 정리**: 최근 7일 기여 기록을 도메인 특화 관점으로 구조화
- **AI 일지 만들기**: 정리 내용 + 킥오프 역할 + 마일스톤 + 집계 건강도를 참조하여 공식 팀 일지 자동 생성
- 생성된 일지는 AI 보관함에 자동 저장 + 팀원 전원 알림

**[ 기능 4 ] 팀 채팅 + AI 회의록**

- 실시간 팀 채팅 (Supabase Realtime)
- **AI 요약**: 채팅 내용을 4~6문장으로 요약하여 채팅창 내 표시
- **AI 회의록**: 도메인 특화 형식(논의·결정·Action Items)의 공식 회의록 생성 및 보관함 저장

**[ 기능 5 ] AI 작업물 보관함**

킥오프·집계·회의록·팀 일지 등 모든 AI 생성물을 프로젝트 단위로 아카이브. 타입별 필터링 및 내용 열람 지원.

**[ 기능 6 ] 마일스톤 관리 + AI 다음 마일스톤**

주차별 마일스톤 태스크 체크, 진행률 자동 계산. "AI로 다음 마일스톤 생성" 버튼으로 현재 진행 상황과 채팅을 분석해 다음 단계 자동 제안.

**[ 공통 ] 공동 목표 대시보드 + 알림 + DM**

- 팀 전체가 실시간으로 공유하는 단일 화면
- 친구 시스템 + 1:1 개인 메시지(파일 첨부 포함)
- 친구 요청·팀원 참가·AI 결과 완료 등 실시간 알림

### ■ 이 솔루션이 도입되었을 때 기대되는 개선점이 무엇인가요?

| 개선 항목 | 도입 전 | 도입 후 |
|---|---|---|
| 팀 세팅 시간 | 역할 논의에 1~2시간 | AI 킥오프로 10분 이내 완결 |
| 무임승차 억제 | 측정 수단 없음 | 일지 기록·AI 집계로 기여도 투명 공유 |
| 전문성 반영 | 일반적 역할 분배 | 도메인 페르소나로 분야 특화 설계 |
| 기록 부담 | 회의록·일지 수동 작성 | AI가 채팅·기여를 자동 정리·보관 |
| 툴 진입 장벽 | 세팅에 30분 이상 | 정보 입력 후 3분 안에 프로젝트 시작 |

## 2. AI 활용 전략

### ■ 이 프로젝트에서 사용할 AI 도구와 모델은 무엇이며, 선택한 이유는 무엇인가요?

**[ 개발 단계 도구 ]**

| 도구 | 용도 | 선택 이유 |
|---|---|---|
| Claude (claude.ai) | 기획 검증, 시장 조사, 프롬프트 설계, 문서 초안 | 구조적 사고와 장문 문서 생성에 강점 |
| Claude Code | 전체 코드베이스 구현 (컴포넌트, API, DB, 스타일링) | 코드베이스 전체 컨텍스트를 파악하고 일관된 구현 가능 |
| Cursor | 로컬 코드 탐색, 빠른 수정 | 파일 기반 편집에 효율적 |

**[ 서비스 내 AI 모델 — Gemini 2.5 Flash ]**

선택 근거:
- 무료 API(250 RPD)로 공모전 데모 수준의 완성도 달성 가능
- 1M 토큰 컨텍스트 윈도우로 팀 전체 프로젝트 맥락을 단일 호출에 포함 가능
- JSON 출력 강제(`responseMimeType: "application/json"`)로 안정적인 구조화 출력
- 고비용 모델 없이 정교한 프롬프트 설계만으로 동등한 결과를 내는 것 자체가 AI 효율적 활용의 증거

---

### ■ 각 AI 도구별 활용 전략을 작성해주세요.

**[ Claude Code — 구현 에이전트 전략 ]**

- 페르소나 설정: 풀스택 개발자. 자연어 요구사항을 코드로 변환
- 활용 범위: React 컴포넌트, Supabase 연동, API 라우팅, 반응형 UI, CSS 애니메이션
- 제약 사항: 보안 관련 로직(API 키 관리, 인증)은 반드시 직접 검토 후 적용

**[ Gemini 2.5 Flash — 6개 에이전트 구성 ]**

| 에이전트 | 호출 시점 | 핵심 출력 |
|---|---|---|
| 킥오프 설계자 | 최초 생성 + 재실행 | 역할 배정 + 마일스톤 (JSON) |
| 다음 마일스톤 생성 | "AI 마일스톤 생성" 버튼 | 다음 주차 마일스톤 1개 (JSON) |
| AI 내용 정리 | "내용 정리" 버튼 | 기여 기록 구조화 요약 (마크다운) |
| AI 일지 만들기 | "일지 만들기" 버튼 | 공식 팀 일지 (마크다운) |
| 채팅 요약·회의록 | "AI 요약" / "회의록" 버튼 | 요약 텍스트 / 공식 회의록 (마크다운) |
| AI 집계 에이전트 | "집계" 버튼 | 건강도·기여도·리스크·우선순위 (JSON) |

**데이터 흐름:**
```
사용자 버튼 클릭
    → 프론트엔드
    → Next.js API 라우트 (API 키 보호, 도메인 페르소나 조립)
    → Gemini API
    → JSON/텍스트 파싱·검증
    → Supabase DB 저장 + 팀원 알림
    → 프론트엔드 렌더링
```

### ■ 토큰 낭비를 최소화하고 유지보수성 및 재현성을 높이기 위한 전략이 있다면 작성해주세요.

**[ 토큰 최소화 ]**

- JSON 출력 강제: `responseMimeType: "application/json"` 설정으로 불필요한 자연어 설명 제거
- 선택적 호출 설계: 모든 AI 기능은 사용자 명시적 액션에만 호출. 자동 트리거 없음
- 컨텍스트 압축: 기여 로그는 집계 요약 형태로 전달, 채팅은 최근 N건만 포함

**[ 유지보수성 ]**

- 프롬프트 파일 분리: system prompt를 `/prompts/` 폴더에 `.txt` 파일로 버전 관리 (kickoff-agent v1→v2→v3)
- 페르소나 중앙화: `src/lib/projectPersona.js` 단일 파일에서 9개 도메인 감지 로직 관리
- 모델 폴백: Gemini 2.5 Flash → 2.0 Flash → 1.5 Flash 자동 폴백으로 API 장애 대응

**[ 재현성 ]**

- temperature 0.2~0.35 고정: 일관성 있는 구조화 출력 우선
- 사실 기반 출력 강제: system prompt에 "기록에 없는 내용 추론 금지" 명시
- 응답 검증: JSON 파싱 실패 시 정규식 블록 추출 재시도 후 명시적 에러 반환
